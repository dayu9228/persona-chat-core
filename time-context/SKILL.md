# 时间感知子技能 (Time Context)

不直接与用户交互。每次被调用时，根据系统当前时间输出一个结构化时间上下文，供核心框架消费。

---

## 输出格式

```
时段: [清晨/上午/下午/傍晚/深夜/凌晨]
星期: [工作日/周末]
敏感度修正: [+0 / +0.5 / +1]
基调建议: [积极/平和/关怀/安静]
```

---

## 时段判定

| 时间范围 | 时段 | 情绪特点 | 敏感度修正 |
|----------|------|----------|------------|
| 04:00-07:00 | 清晨 | 刚醒或通宵，情绪脆弱期 | +0.5 |
| 07:00-12:00 | 上午 | 精力充沛，适合推进任务 | +0 |
| 12:00-17:00 | 下午 | 平稳期，可能疲劳累积 | +0 |
| 17:00-20:00 | 傍晚 | 过渡期，容易思绪起伏 | +0.5 |
| 20:00-22:00 | 晚间 | 放松状态，容易敞开心扉 | +0.5 |
| 22:00-04:00 | 深夜 | 情绪易放大，emo高发区 | +1 |

---

## 星期判定

- **工作日（周一至周五）**：基调平和，白天的疲惫和焦虑可能累积
- **周末（周六、周日）**：基调放松，但独处时间增加可能触发孤独感
- **节假日前夜**：基调偏兴奋或放松

---

## 季节判定（可选）

| 季节 | 情绪提示 |
|------|----------|
| 春（3-5月） | 万物复苏，适合开启新计划 |
| 夏（6-8月） | 燥热易怒，注意情绪管理 |
| 秋（9-11月） | 容易感伤，但也适合沉淀 |
| 冬（12-2月） | 昼短夜长，需要更多温暖 |

---

## 消费方式

核心框架（persona-chat-core）读取本技能输出的时间上下文后：

1. **敏感度修正值**直接叠加到情绪研判的综合分数上
2. **基调建议**影响最终回应的语气底色
3. **时段信息**辅助维度3（行为-上下文特征）的判断

> 本技能不储存任何用户数据。每次调用独立运行，无状态残留。
