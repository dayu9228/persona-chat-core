# v1.2 — 完整决策链

## 这个版本做了什么

v1.2 是第一次结构级质变。v1.0 是说明书，v1.1 是公式，v1.2 是完整决策链。

### 核心改动：研判从模糊权重变成精准流水线

v1.1 的五维研判是一张权重表——每个维度给一个系数，最后加总。这有两个问题：一是维度之间会互相稀释（词汇信号强但句式信号弱，本该锁定焦虑结果被平均成中性的情况）；二是不同输入类型（打字、语音、图片）用同一把尺子，精度打折扣。

v1.2 的解决方案：**选门 → 定强度 → 锁类型**。

**选门**：先判断用户输入类型，走对应的优先级链。打字用户词汇优先，语音用户句式优先，发图用户分享行为优先。主导维度锁定方向，后续维度只做验证不推翻。

**定强度**：信号分两级（明确信号和弱信号），数量定档。≥3 个明确信号是高，1-2 个或 3 个弱信号含 1 明确是中，仅弱信号是低，全干净是静默。

**锁类型**：11 种情绪各有排他锚点。焦虑和兴奋都有反问，但焦虑的反问是求解的、兴奋的是修辞性的。后悔和比较焦虑都很痛苦，但后悔是回看自己过去的选择，比较焦虑是看别人现在的状态。平局链按误判代价兜底——愤怒被忽视代价最高，排在链头。

### 新增：混合消息协议

v1.1 面对"我心态崩了，你帮我看看这个bug"这类消息时很尴尬：全走情绪太重，全走技术太冷。v1.2 用四层压缩解决：每层锚定 1-2 句，四层全保留，宁短不缺层。不丢情绪也不躲问题。

### 新增：长上下文锚点

实测发现上下文超过 100K token 后模型开始产生轻微幻觉。之前的设计是退化框架（缩成一句话），实践证明不行——退化了也挡不住幻觉，反而牺牲体验。

v1.2 的方案：不退化，不降级。80K 起埋 HTML 注释锚点追踪研判状态。锚点本身是里程标记——模型看到历史中有锚点即知已过 80K。当锚点持续存在多轮、对话深度明显增加时（150-200K 区间），主动提醒用户开新窗口。用户选择继续，维持完整四层不做任何降级。

### 新增：思考链透明化

用户可以打开看 AI 每一步的研判路径。高/中情绪完整展示六行（五维研判→心理归因→Vibe→Sparks→Reflections→Will），低情绪折叠为单行摘要，静默模式不展示。上下文无关，不因对话变长而降级。

### 扩展：归因表

从 v1.1 的 7 种情绪扩展至 11 种。新增愤怒（边界感知+无力感+累积效应三层叠加）、孤独（连接渴望落空+自我叙事中断+意义锚点松动）、后悔（反事实思维的两层结构）、沉默退缩（自保/思考中/习得性沉默三种可能）的独立心理学归因模型。

### 修了哪些洞

v1.1 深夜漏洞审查发现的 9 个问题全部修复：
- 铁律与防疲劳冲突 → 防疲劳模式下 Vibe 以一句话形式存在，视为已满足铁律
- 归因表缺愤怒和孤独 → 已补充独立归因模型
- 静默模式边界模糊 → 三重同时满足条件，精确触发
- 长上下文 50K+ 后 MOOD 框架消失 → 80K 锚点机制
- time-context 时段和基调未被消费 → 全面接入研判和回应流程
- 五维度量化路径 → 信号分级+数量定档
- 极短消息分支 → 单字/双字跳过展开规则
- 敏感度修正硬顶 → +1.5
- philosophy-engine 分类对齐 → 困境类型直接从研判类型标签映射
